Treinamento IA / Módulo 14 — Deep Thinking
14

Deep Thinking e Reasoning Models

Test-time compute, RLVR, chain-of-thought emergente. Como o1, Claude extended thinking, DeepSeek-R1 e Gemini 2.5 mudam a economia da IA.

Seção 14.1

Por que reasoning models existem

Até ~2024, o caminho para melhorar LLMs era escalar o pré-treinamento. Mas dados de alta qualidade são finitos, custo marginal explodiu, e muitas tarefas se beneficiam mais de pensar mais na hora da resposta do que de um modelo maior.

A resposta: test-time compute scaling — deixar o modelo pensar mais durante a inferência.

📚
Paper seminal

Snell, C. et al. (2024). "Scaling LLM Test-Time Compute Optimally can be More Effective than Scaling Model Parameters". arXiv:2408.03314. arxiv.org/abs/2408.03314

Seção 14.2

Como um reasoning model funciona

O modelo gera uma longa chain-of-thought interna (às vezes milhares de tokens) antes de produzir a resposta final. Esses thinking tokens são treinados via reinforcement learning com recompensas verificáveis.

  1. Base model — LLM pré-treinada normalmente
  2. CoT prompting — força pensamento passo a passo
  3. RLVR (RL from Verifiable Rewards) — tarefas com resposta checável (matemática, código com testes, lógica). +1 se acerta, 0 se erra.
  4. Emergência de comportamentos: auto-correção, backtracking, verificação, planejamento. Não foram programados — emergem do RL.
📚
Paper obrigatório

DeepSeek-AI (2025). "DeepSeek-R1: Incentivizing Reasoning Capability in LLMs via RL". arXiv:2501.12948. arxiv.org/abs/2501.12948 — demonstra que RL puro (sem SFT inicial) já induz reasoning emergente.

OpenAI (2024). "Learning to Reason with LLMs" (post técnico do o1). openai.com

Kimi Team (2025). "Kimi k1.5: Scaling RL with LLMs". arXiv:2501.12599

Seção 14.4

Quando usar thinking

Use reasoning quando
  • Tarefa é multi-passo e cada passo gera erro composto (matemática, prova, debugging)
  • Resposta errada custa caro (código de produção, análise legal)
  • Você quer o melhor possível e pode pagar 5–20× mais tokens
Não use quando
  • Tarefa é de recuperação/formatação (extrair JSON, traduzir, resumir)
  • Latência é crítica (chat conversacional, autocomplete)
  • O modelo base já acerta — meça antes de pagar thinking

Evidência: modelos sofrem de overthinking em tarefas fáceis. Ver Chen et al. 2024 — "Do NOT Think That Much for 2+3=?", arXiv:2412.21187.

Seção 14.5

Implicações para programação

  • Agentes de codificação (Claude Code, Cursor Agent, Devin) combinam thinking + tool use + loops. O thinking decide o que fazer a seguir.
  • SWE-Bench Verified saltou de ~20% (2024) para >70% (reasoning + agentes, 2025).
  • Custo real = input + output + (thinking × várias chamadas em loop). Monitore.
Seção 14.6

Limitações honestas

  • Ainda alucinam e erram matemática simples ocasionalmente.
  • Faithfulness of CoT: Turpin et al. 2023 (arXiv:2305.04388) — a CoT mostrada nem sempre reflete o que o modelo "realmente" computou. Anthropic 2025 confirma em modelos novos.
  • Thinking injection é uma nova superfície de ataque.
