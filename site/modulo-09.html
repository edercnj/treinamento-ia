Seção 10.1

Skills: Expandindo as Capacidades

Skills são comandos especializados que estendem o que uma ferramenta de IA pode fazer. No contexto do Claude Code, skills são invocadas via comandos / (slash commands) e encapsulam workflows complexos em uma única ação.

Analogia

Skills são como macros ou scripts reutilizáveis — em vez de dar instruções detalhadas toda vez, você invoca um comando que já sabe o que fazer.

Exemplos de Skills no Claude Code

Skill Comando O que faz
Commit /commit Analisa mudanças, gera mensagem de commit seguindo convenções
Code Review /review Revisa código de um PR com checklist completo
Simplify /simplify Revisa código alterado buscando simplificações
Claude MD Improver /claude-md-improver Audita e melhora CLAUDE.md do projeto

Como Skills funcionam internamente

Usuário digita: /commit
1. Claude Code intercepta o comando
2. Carrega a definição da skill
Prompt especializado + regras + ferramentas permitidas
3. Executa o workflow definido
a. git status (ver mudanças)
b. git diff (ler as mudanças)
c. git log (ver estilo de commits anteriores)
d. Analisa e gera mensagem de commit
e. Cria o commit
4. Retorna resultado ao usuário

Criando Skills Customizadas

Skills customizadas podem ser definidas como prompts reutilizáveis. No ecossistema Claude Code, isso é feito através de Agent Skills — recursos baseados em filesystem que fornecem expertise específica de domínio via arquivos SKILL.md.

10.1.1 Template Completo de uma Skill

Para criar uma skill customizada no Claude Code, crie o arquivo .claude/skills/nome-da-skill/SKILL.md com o seguinte formato:

.claude/skills/nome-da-skill/SKILL.md---
name: nome-da-skill
description: O que essa skill faz (frase curta que o modelo usa para decidir quando invocar)
---

# Instruções da Skill

Quando invocada, você deve:
1. Analisar o contexto atual (arquivos abertos, mudanças no git, etc.)
2. Executar os passos específicos desta skill
3. Reportar o resultado ao usuário seguindo o formato definido abaixo

## Regras

- Use sempre Português para mensagens de usuário
- Prefira ferramentas já disponíveis antes de instalar novas dependências
- Nunca execute comandos destrutivos sem confirmação

## Formato de saída

Ao terminar, forneça um resumo em 2-3 frases com:
- O que foi feito
- Arquivos afetados
- Próximos passos sugeridos

Exemplo real — uma skill para gerar testes unitários:

.claude/skills/gerar-testes/SKILL.md---
name: gerar-testes
description: Gera testes unitários para o arquivo atual seguindo o framework do projeto (pytest, jest, etc.)
---

# Gerar Testes Unitários

Quando invocada:

1. Identifique o arquivo alvo a partir do contexto do usuário
2. Detecte o framework de teste lendo package.json, pyproject.toml ou similar
3. Para cada função pública / método exportado:
   - Caso feliz (happy path)
   - Caso de borda (inputs vazios, null, valores extremos)
   - Caso de erro (exceções esperadas)
4. Salve os testes em `tests/` ou `__tests__/` seguindo convenção existente
5. Rode o test suite e ajuste até todos passarem

## Não fazer

- Não mocke dependências reais sem necessidade
- Não edite o arquivo original para forçar testabilidade sem discutir antes
💡
Dica

Execute /help dentro do Claude Code para listar todas as skills disponíveis (built-in, de plugins e customizadas do projeto).

10.1.2 Equivalentes em Outras Ferramentas

O conceito de Skills é principalmente um diferencial do Claude Code. Outras ferramentas têm features análogas, mas com menor flexibilidade:

  • GitHub Copilot — Slash Commands:
    • Built-in: /explain, /fix, /tests, /doc, /review
    • Custom: via .github/copilot-instructions.md
    • Copilot Extensions para extensibilidade via GitHub Apps
    • Referência: docs.github.com/en/copilot/how-tos/custom-instructions-and-extensions
  • Cursor — Custom Commands e @ References:
    • Comandos @ para referenciar arquivos, docs, web, git
    • Custom modes em .cursor/modes/ (ex.: architect mode, debugger mode)
    • Rules para criar comportamento reutilizável
    • Referência: cursor.com/docs
  • OpenAI Codex — Prompts Salvos e Profiles:
    • Profiles configurados via TOML
    • Saved prompts / presets reutilizáveis
    • Referência: developers.openai.com/codex
  • Windsurf — Workflows:
    • Workflows encapsulam sequências de ações automatizadas
    • Referência: docs.windsurf.com
Ferramenta Nome do Conceito Localização / Arquivo Exemplos built-in
Claude Code Agent Skills .claude/skills/*/SKILL.md /commit, /review, /simplify
GitHub Copilot Slash Commands + Extensions .github/copilot-instructions.md /explain, /fix, /tests
Cursor Custom Modes + @ References .cursor/modes/ @Docs, @Web, @Git
OpenAI Codex Profiles + Saved Prompts TOML config Profiles nomeados por tarefa
Windsurf Workflows Configuração de projeto Workflows de build/deploy

10.1.3 Problemas que Skills Resolvem

# Problema Como skills resolvem
1 Repetição de instruções complexas Encapsula um prompt longo em um comando curto (/commit em vez de 20 linhas de instruções)
2 Workflows padronizados Commit, review, deploy seguem o mesmo processo independente de quem invoca
3 Abstração de tool calls complexos Esconde a orquestração de múltiplas ferramentas (git, tests, lint) atrás de um único comando
4 Onboarding de novos devs Skills prontas permitem que novos membros executem workflows do time sem precisar aprender tudo
5 Consistência entre membros do time Todos invocam a mesma skill, eliminando variação por "jeitinho" de cada um
6 Documentação viva A skill é a documentação executável — ela não fica desatualizada porque é usada
7 Redução de erros humanos Passos críticos (ex.: rodar testes antes de commit) não são esquecidos
8 Compartilhamento entre projetos Skills podem ser versionadas em repositórios e reutilizadas em múltiplos projetos

10.1.4 Quando Criar uma Skill (vs. Prompt Direto)

Crie uma skill quando...
  • A ação é repetitiva (acontece toda semana / toda sprint)
  • O workflow tem mais de 3 passos encadeados
  • Diferentes pessoas do time precisam executar a mesma tarefa
  • Os passos exigem contexto específico do projeto (convenções, ferramentas internas)
  • Você quer padronizar a saída (formato de commit, relatório, etc.)
  • Existe risco de esquecer um passo crítico
NÃO crie uma skill quando...
  • A ação é única ou pontual (uso one-shot)
  • Depende muito de contexto efêmero (ex.: debugging de um bug específico)
  • É trivial (pode ser resolvido com um prompt de 1 linha)
  • Ainda está sendo descoberta / estabilizada — espere o workflow amadurecer
  • Requer decisões humanas complexas em cada execução
