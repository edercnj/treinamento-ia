Seção 16.2

Limitações dos benchmarks

  1. Data contamination — benchmarks vazam no treinamento. HumanEval em particular está saturado. Ver Balloccu 2024, arXiv:2402.03927.
  2. Gaming — laboratórios otimizam para o leaderboard.
  3. Single-task / single-turn — não capturam comportamento agêntico.
  4. Viés linguístico — quase todos em inglês. Para PT-BR: ENEM-Challenge, BLUEX.
  5. Saturação — MMLU passou 85% (humano expert) em 2024. Novos benchmarks (HLE — Humanity's Last Exam, 2025) existem por isso.
Seção 16.3

Como montar sua própria eval

Regra #1: a melhor eval é a mais próxima do seu caso de uso real.

Framework mínimo
  1. Golden set — 30–100 exemplos representativos com resposta esperada.
  2. Métrica — exact match, LLM-as-judge (com cuidado, Zheng 2023 arXiv:2306.05685) ou humano.
  3. Regressão — rode em toda mudança de modelo/prompt.
  4. Adversarial set — casos que historicamente falharam.

Ferramentas:

  • Inspect (UK AISI) — inspect.aisi.org.uk
  • OpenAI Evals — github.com/openai/evals
  • promptfoo — promptfoo.dev (open source)
  • Braintrust, LangSmith, Langfuse, Helicone (pagas)
Seção 16.4

Leaderboards úteis (com ceticismo)

  • Artificial Analysis — comparativo de qualidade, velocidade, preço
  • Chatbot Arena / LMArena — preferência humana
  • Aider LLM Leaderboard — edição de código real
  • SWE-Bench Leaderboard
