Embeddings, Retrieval e RAG Avançado

Dense/sparse/hybrid retrieval, re-rankers, GraphRAG, Agentic RAG, Contextual Retrieval. Além do RAG ingênuo.

Seção 17.2

Recuperação: dense, sparse, hybrid

  • Dense retrieval (DPR) — produto escalar de embeddings. Karpukhin 2020, arXiv:2004.04906.
  • Sparse retrieval (BM25) — TF-IDF tradicional; imbatível em exact match e nomes próprios.
  • Hybrid — combina ambos via reciprocal rank fusion (Cormack 2009).
  • Late interaction (ColBERT) — Khattab & Zaharia 2020, arXiv:2004.12832.
  • Re-rankers — segundo estágio; cross-encoders (BGE reranker, Cohere rerank).
Seção 17.3

Anatomia de um RAG de produção

query
  → query rewriting (LLM expande, decompõe)
  → retrieval (hybrid: BM25 + dense)
  → re-ranking (cross-encoder)
  → context packing (dedup, compressão)
  → generation (LLM com citações forçadas)
  → verification (groundedness check)

Cada caixa é um ponto de falha. Avalie cada estágio separadamente.

Seção 17.4

RAG vs. long-context vs. fine-tuning

EixoRAGLong-contextFine-tuning
Atualização de dadosBaratoBaratoCaro
VolumeIlimitadoLimitado à janelaIlimitado no treino
CitabilidadeNativaDifícilImpossível
Latência/queryMédiaAltaBaixa
Custo/queryBaixoAltoBaixo
Estilo/formatoFracoVia few-shotForte

Heurística: dados mutáveis e auditáveis → RAG. Estilo consistente → fine-tune. One-shot com documento específico → long-context.

Seção 17.5

Estado da arte (2025–2026)

  • GraphRAG — Microsoft (Edge et al. 2024, arXiv:2404.16130). Knowledge graph extraído do corpus; queries globais tratáveis.
  • Agentic RAG — o LLM decide quando e o que buscar em loop (Search-R1, DeepSeek-R1 variants).
  • Late chunking — Jina AI 2024. Embed do documento inteiro, pool por chunk — preserva contexto cross-chunk.
  • Contextual retrieval — Anthropic 2024. Prefixa cada chunk com resumo do documento antes de embedar.
Seção 17.6

Vector stores em 2026

  • pgvector (Postgres) — default pragmático
  • Qdrant, Weaviate, Milvus, LanceDB — especializados
  • Turbopuffer, Pinecone — serverless hospedado
  • FAISS / HNSWLib — bibliotecas de ANN, não DBs
